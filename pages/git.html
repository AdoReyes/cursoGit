Documentación GitHub

¿Para qué usar GIT?

  1. Permite regresar a versiones anteriores de forma sencilla y muy rápida.
  2. Facilita el trabajo colaborativo.
  3. Permite respaldar tus proyectos en la nube (ej con github).
  4. Reduce considerablemente los tiempos de deploy.
  5. Las "branches" o ramas, permiten trabajar con una base de código paralela al proyecto en sí.

Comandos básicos

Versión de Git

git version

Registrar nuevo usuario asociado a git:

git config --global user.name "mi nombre"

git config --global user.email "myemail@example.com"

Ver los datos del usuario logueado

git config user.name

git config user.email

Iniciar un repositorio local

git init

Ver estatus de los archivos no registrados

git status -s

Agregar todos los cambios no registrados (pendientes)

git add .

Realizar commit de los cambios localmente

git commit -m "Comentario del commit"

Ver todos los commits del más reciente al más antiguo

git log --oneline

Bug de commit: la consola no responde presionar:

:q!

Repositorio remoto

Crear una carpeta local

Nombre de la nueva carpeta vacía

Conectarse al repositorio remoto

git remote add origin "IP de la ruta del respositorio

Hacer push (Subir los cambios al respositorio).

git push -u origin master

Omitir archivos en GitHub

.gitignore

arhivo.js // Ignora el archivo en cuestion

*.js // Ignora todos los arhivos con extensión .js

node_modules/ //Ignora toda la carpeta

Historial de commits

Ver historial

git log --oneline

git checkout IdDelCommit

Volver a la versión actual

git checkout nombreDeLaRama o IdActualCommit

Resetear un commit

Resetear un commit y volver a un commit especifico

git reset —hard “IdDelCommit”

Recuperar o resetear un commit

Recuperar los commits despues de un reseteo

git reflog

git reset —hard “IdDelCommitQueSeQuiereRecuperar”

Bajar los cambios del repositorio en la nube

git pull origin “NombreDeLaRama”

Quitar un commit específico

git revert “IdDelCommit” //Es ncesario hacer commit nuevamente

Trabajar con ramas

Rama donde nos encontramos actualmente

git branch

Crear una nueva rama

git branch nombreDeLaNuevaRama

git branch //Verificar que la rama se creo

Pasarnos a la nueva rama creada

git checkout nombreRama

Ver el organigrama de las ramas

git log --oneline --graph

Hacer push en la rama creada

git push --set-upstream origin nombreRama

Fusionar una rama con la master

Posicionarse dentro de la rama master (es un decir) y ejecutar:

git merge nombreRama

git log --oneline --graph

Eliminar una rama

git branch -d nombreRama

Resolver conflictos

Seleccionar el commit correcto

git add .

git commit -m "descripción del conflicto"
